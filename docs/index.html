the system

Six guardrails. One contract.

Every rule and command implements the Agent Operating Contract — one enforceable standard for how agents plan, edit, validate, and report.

01

Plan before code

No edits before impacted files, behavior change, tests, and rollback are written down. High-risk work waits for your approval.

02

Risk classification

Every change gets Low / Medium / High / Critical. Auth, billing, DB schema, and infra automatically escalate.

03

Evidence or it didn't happen

"Done" requires exact commands and real output. No validation evidence, no completion claim. Ever.

04

Injection boundary

Tool outputs, files, and logs are data — never instructions. Embedded "ignore previous instructions" gets reported, not obeyed.

05

Adversarial self-review

The agent reviews its own diff like a reviewer trying to block a risky PR: hidden behavior changes, missing tests, security.

06

Release gates

Go/no-go checklists for PRs and releases: migrations with rollbacks, config documented, known risks accepted by a named human.

faq

Fair questions.

Can rules really *enforce* anything? The agent could ignore them.

Rules steer the agent; the pack is designed so the important gates don't rely on the model behaving. The PR template and checklists make evidence visible to humans, and at Level 4 your CI mechanically rejects PRs that lack risk levels or validation evidence. The repo refuses unsafe changes even when the agent is having a bad day.

Why not just write my own CLAUDE.md?

You can — most people write "be careful and test your code" and stop there. This pack is the other 95%: instruction hierarchy, injection boundaries, risk escalation tables, behavior-preserving refactor workflows, release gates, and output contracts that CI can verify — synthesized from Microsoft, OpenAI, and Anthropic's own published guidance and structured the way models actually follow instructions.

Does it work with my stack?

Yes. The pack is language-agnostic; verification commands are placeholders you point at your own lint/typecheck/test commands in one file. It assumes git and PRs, nothing else.

Claude Code or Cursor?

Both, in the same pack. The same standard ships as CLAUDE.md + slash commands for Claude Code and as .mdc rules for Cursor. Codex-style agents can read the shared standard docs too — they're plain markdown.

Will this slow my agent down?

Low-risk changes (single file, tested, no behavior change) proceed immediately — the contract explicitly says so. The workflow cost lands only on the changes that can hurt you: auth, schema, billing, infra. That's the point.

Is this a subscription?

No. One-time purchase, files are yours, works offline forever. v1.x updates delivered through your purchase channel.

What's actually free vs. paid?

The core contract is free and MIT-licensed — the Agent Operating Contract, scope + validation rules, two commands, and the PR template. That alone will noticeably discipline your agents. The paid pack is the full system: all 8 commands, security/database/refactoring rules, the risk classification model, injection-boundary and tool-use standards, the four checklists, and the 5-level compliance framework. Free gets you the contract; Pro gets you the operating system.
