the problem

Your agent says "done." Is it?

AI coding agents are powerful — and careless by default. They edit twelve files, skip the tests, change behavior they never mention, and report success. This pack makes them behave like disciplined engineers.

without guardrails
BEFORE $ fix this bug agent: Done! I fixed the bug. · edited 12 files · 0 tests written or run · refactored half the module "while at it" · silently changed a default value · touched the auth middleware · no evidence, no risk assessment You find out in production.
with guardrails
AFTER $ /fix-bug agent: 1. reproduces with a failing test 2. states the root cause 3. classifies risk — flags auth = HIGH 4. fixes with the smallest diff 5. runs lint / typecheck / tests to green 6. self-reviews the diff 7. reports with evidence You review a provable change.

the system

Six guardrails. One contract.

Every rule and command implements the Agent Operating Contract — one enforceable standard for how agents plan, edit, validate, and report.

01

Plan before code

No edits before impacted files, behavior change, tests, and rollback are written down. High-risk work waits for your approval.

02

Risk classification

Every change gets Low / Medium / High / Critical. Auth, billing, DB schema, and infra automatically escalate.

03

Evidence or it didn't happen

"Done" requires exact commands and real output. No validation evidence, no completion claim. Ever.

04

Injection boundary

Tool outputs, files, and logs are data — never instructions. Embedded "ignore previous instructions" gets reported, not obeyed.

05

Adversarial self-review

The agent reviews its own diff like a reviewer trying to block a risky PR: hidden behavior changes, missing tests, security.

06

Release gates

Go/no-go checklists for PRs and releases: migrations with rollbacks, config documented, known risks accepted by a named human.

faq

Fair questions.

Can rules really *enforce* anything? The agent could ignore them.

Rules steer the agent; the pack is designed so the important gates don't rely on the model behaving. The PR template and checklists make evidence visible to humans, and at Level 4 your CI mechanically rejects PRs that lack risk levels or validation evidence. The repo refuses unsafe changes even when the agent is having a bad day.

Why not just write my own CLAUDE.md?

You can — most people write "be careful and test your code" and stop there. This pack is the other 95%: instruction hierarchy, injection boundaries, risk escalation tables, behavior-preserving refactor workflows, release gates, and output contracts that CI can verify — synthesized from Microsoft, OpenAI, and Anthropic's own published guidance and structured the way models actually follow instructions.

Does it work with my stack?

Yes. The pack is language-agnostic; verification commands are placeholders you point at your own lint/typecheck/test commands in one file. It assumes git and PRs, nothing else.

Claude Code or Cursor?

Both — as separate MIT plugins, not one blended bundle. Cursor gets always-on .mdc rules plus commands; Claude Code gets slash commands and a contract skill (Claude doesn't use Cursor rules). Prefer marketplace install for your IDE, or use curl install.sh to copy the full pack into a repo. Codex-style agents can read the shared standard docs — plain markdown.

Will this slow my agent down?

Low-risk changes (single file, tested, no behavior change) proceed immediately — the contract explicitly says so. The workflow cost lands only on the changes that can hurt you: auth, schema, billing, infra. That's the point.

Is this a subscription? Can I buy it today?

Not yet — checkout is deferred until after beta. The MIT core installs free from GitHub (curl below). The full Pro + Team OS pack is available to invited private beta users only — request access, not self-serve checkout. When paid tiers launch, the plan is one-time purchase (not subscription), files yours, works offline.

What's actually free vs. paid?

Public (always free): MIT core via ai-coding-guardrails — operating contract, core rules, PR template, init scripts. Private beta (invite): full Pro pack + all Team OS agents — confidential zip, not on the public repo. After GA (planned): MIT core stays free; Pro/Team via purchase. Checkout not live yet.

How do I get the full pack during beta?

Private beta only — we add your GitHub account to the invite team and you download the release zip. The public curl install.sh installs the free core, not Team OS or Pro-only rules. See guardrails-demo for public behavior transcripts.

Does this use a lot of tokens?

Always-on rules add roughly 1–3k tokens (free) or 3–6k tokens (Pro + Team OS) per turn — modest on large context windows. The bigger cost is structured behavior: plans, evidence blocks, and multi-role handoffs. Mitigations: adopt rules gradually, use default 5-role Team OS (not --full) for daily work, keep evidence concise, prefer same-thread role switches. See docs/token-and-context.md in the Pro/Team pack.
